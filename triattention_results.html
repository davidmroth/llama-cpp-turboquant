How It Works

1. Self-Calibrate
Hooks Q projections via cb_eval during warm-up. Accumulates frequency-domain stats across all 28 layers. Adaptive EMA updates continuously. Zero external dependencies.
2. Score
Trigonometric scoring per arXiv:2604.04921. Computes Q·conj(K) in frequency domain with cos/sin phase expansion. Identifies which tokens the model will attend to. CPU, <1ms.
3. Evict
Every 128 tokens, score all cache entries, sort by importance, evict highest-scoring (least important) tokens. Protect recent window. Cache stays bounded at budget.
Key finding during development: The trig scoring formula assigns high scores to unimportant tokens (measuring orthogonality to Q center, not alignment). Early versions had the sort inverted — random eviction outperformed trig scoring until this was caught via A/B testing. With the correct sort order, the scoring cleanly separates important from unimportant tokens.
